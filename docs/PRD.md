# 인형공예 작품전시 블로그 - PRD (Product Requirements Document)

## 1. 개요

**프로젝트명**: 인형공예 작품전시 블로그
**목적**: Notion을 CMS로 활용하여 인형공예 작품을 전시하고 관리하는 현대적인 블로그 플랫폼 구축
**타겟 사용자**: 인형공예 작품 감상자, 작품 판매 고객, 작품 제작자
**배포 환경**: Vercel (Next.js 최적화)

---

## 2. 목표 및 목적

### 주요 목표
1. **콘텐츠 관리 효율화**: Notion을 CMS로 사용하여 작품 정보 관리
2. **자동 동기화**: Notion에서 글 작성하면 블로그에 자동으로 반영
3. **사용자 경험 최적화**: 반응형 디자인과 직관적인 네비게이션 제공
4. **검색 및 필터링**: 카테고리별 필터링과 검색 기능으로 작품 발견 용이

### CMS 선택 이유
- **비용 효율성**: 별도의 복잡한 백엔드 서버 불필요
- **사용 편의성**: 비개발자도 쉽게 콘텐츠 관리 가능 (Notion UI)
- **무료/저비용**: Notion Free 플랜으로 시작 가능
- **자유로운 확장성**: 데이터베이스 필드 자유롭게 추가 가능

---

## 3. 주요 기능

### 3.1 필수 기능 (MVP)
- **글 목록 페이지**: 최근 발행된 작품 목록 표시 (타일/리스트 뷰)
- **글 상세 페이지**: 개별 작품 정보 및 이미지 전시
- **카테고리 필터링**: 작품 종류(예: 소품, 의류, 소품 등)별 필터링
- **반응형 디자인**: 모바일, 태블릿, 데스크탑 모두 최적화
- **빠른 로딩**: 정적 생성(SSG) 및 증분 정적 재생성(ISR)

### 3.2 추가 기능 (향후)
- 검색 기능 (제목, 카테고리 기반)
- 댓글/후기 시스템
- 관련 작품 추천
- 이미지 갤러리 모드
- SNS 공유 기능
- 이메일 구독 기능

## 5. Notion 데이터베이스 구조

### 5.1 데이터베이스 필드 정의

| 필드명 | 타입 | 필수 여부 | 설명 |
|--------|------|---------|------|
| **Title** | Title | ✅ 필수 | 작품 제목 |
| **Category** | Select | ✅ 필수 | 작품 카테고리 (예: 소품, 의류, 액세서리) |
| **ImageUrl** | Url | ✅ 필수 | 작품 메인 이미지 URL |
| **Status** | Select | ✅ 필수 | 상태 (초안/발행됨) - 발행된 글만 블로그에 표시 |
| **Description** | Text | - | 작품 짧은 설명 (150자 이내) |
| **Content** | Rich Text | - | 작품 상세 설명 (본문) |
| **PublishedAt** | Date | ✅ 필수 | 발행 날짜 |
| **Materials** | Multi-select | - | 사용된 재료 (예: 펠트, 실, 눈) |
| **Size** | Text | - | 작품 크기/규격 |
| **Price** | Number | - | 판매 가격 (선택사항) |
| **Tags** | Multi-select | - | 추가 태그 |

### 5.2 데이터베이스 설정
- **정렬 기준**: PublishedAt (최신순)
- **필터**: Status = "발행됨" (프로덕션에서만)
- **아이콘**: 🪡 (바늘과 실)

## 6. 화면 구성 및 사용자 흐름

### 6.1 홈 페이지 (/)
- **목적**: 최근 발행된 작품 목록 표시
- **구성요소**:
  - 헤더: 로고, 테마 토글, 네비게이션
  - 히어로 섹션: 프로젝트 소개
  - 작품 그리드: 카드 형식의 작품 목록 (반응형 3열 → 2열 → 1열)
  - 페이지네이션: 페이지당 12개 작품 표시
  - 푸터: 기본 정보, 소셜 링크

### 6.2 글 상세 페이지 (/post/[slug])
- **목적**: 개별 작품 정보 전시
- **구성요소**:
  - 헤더: 작품 제목, 카테고리 배지
  - 메인 이미지: 반응형 이미지 표시
  - 메타 정보: 작품명, 제작자, 발행 날짜
  - 작품 상세 정보: 
    - 설명 (마크다운 형식)
    - 재료 목록
    - 크기/규격
    - 가격 (있는 경우)
  - 네비게이션: 이전/다음 작품 링크

### 6.3 카테고리 페이지 (/category/[category])
- **목적**: 카테고리별 작품 목록 표시
- **구성요소**:
  - 카테고리 헤더
  - 필터링된 작품 그리드
  - 카테고리 설명

### 6.4 레이아웃 구조
```
┌─────────────────────────────────┐
│      고정 네비게이션 바 (navbar)     │
├─────────────────────────────────┤
│                                 │
│       메인 콘텐츠 (page)          │
│      (flex: 1로 최소 높이)       │
│                                 │
├─────────────────────────────────┤
│      고정 푸터 (footer)          │
└─────────────────────────────────┘
```

## 9. 기술 설계 상세

### 9.1 데이터 페칭 전략
```typescript
// lib/notion.ts
- getPublishedPosts(): Promise<Post[]>
- getPostBySlug(slug: string): Promise<Post | null>
- getPostsByCategory(category: string): Promise<Post[]>
- getCategories(): Promise<string[]>
```

### 9.2 정적 생성 전략
- **SSG**: 빌드 타임에 모든 글 페이지 사전 생성
- **ISR**: 새 글 발행 시 최대 60초 내에 자동 재생성
- **Revalidation Tag**: 특정 패턴 기반 재생성

### 9.3 URL 구조
- 홈: `/`
- 글 상세: `/post/[slug]` (예: `/post/my-first-doll`)
- 카테고리: `/category/[category]` (예: `/category/accessories`)

### 9.4 Notion API 성능 최적화
- 데이터베이스 쿼리 캐싱 (ISR 활용)
- 배치 요청 처리
- 불필요한 필드 제외 (필드 선택)

## 11. 제약사항 및 고려사항

### 기술적 제약사항
1. **Notion API 한계**: 동시성 제한 (3 RPS)
2. **이미지 저장소**: Notion 외부 URL 저장 필수 (외부 스토리지 사용)
3. **빌드 시간**: 글이 많을 시 빌드 시간 증가 가능

### 설계 결정사항
| 항목 | 선택 | 이유 |
|------|------|------|
| 인증 | 없음 | MVP 범위에서 공개 블로그 |
| 검색 | 추후 | 초기 글이 적으면 불필요 |
| 댓글 | 추후 | 커뮤니티 필요 시 추가 |
| 배포 | Vercel | Next.js 최적화, 자동 배포 |

### 보안 고려사항
- Notion API 키는 서버에서만 사용 (`.env.local`)
- 공개 데이터베이스만 노출
- 사용자 입력값 없으므로 XSS/CSRF 위험 최소
